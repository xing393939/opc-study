笔记

  1. 防火墙
    1. WAF
    2. Shield
  2. 存储相关：
    1. block storage（容量16T）：
      1. General Purpose SSD （gp2/gp3）：1.6w IOPS，适合随机IO
      2. Provisioned SSD（io1/io2）：6.4w IOPS，适合随机IO，适合数据库
      3. Throughput Optimized HDD （st1）：500 IOPS，适合顺序IO
      4. Cold HDD （sc1）：250 IOPS，适合顺序IO
      5. Instance Store：200w IOPS，关机数据丢失
    2. file storage：
      1. Amazon Elastic File System：50w IOPS，多ec2共享，容量不限
      2. Amazon FSx for Windows：network filesystem for Windows，容量64T
      3. Amazon FSx For Lustre：high-performance，容量不限
    3. object storage：S3，如果出现file storage则不能选S3
      1. Standard
      2. Intelligent：智能
      3. Standard-IA：不经常访问、取回有费用
      4. One Zone-IA：单AZ
      5. Glacier：>90天，取回时间1m~12h
      6. Glacier Deep Archive：>180天，取回时间12h~48h
    4. EBS 的优点：
      1. 自动在当前AZ备份，单block failure后自动替换
      2. 不停机的更换 type, size, IOPS capacity
  3. Auto Scaling Group scaling 策略
    1. simple scaling：When CloudWatch alarm is triggered，有延迟
    2. target tracking scaling：CPU 利用率超过 40%，立即生效
    3. scheduled scaling
  4. Auto Scaling Group terminate 策略
    1. 多AZ选择AZ中ec2最多的
    2. ec2中配置项最老的
    3. ec2中最接近账单周期的
    4. 满足以上条件的ec2随机terminate
  5. 账号集成：
    1. Access AWS resources using on-premises credentials stored in Active Directory ！！！
    2. Amazon Cognito 用户池安全：启用多重验证(MFA)作为二次验证
  6. RDS
    1. When failing over, Amazon RDS points CNAME to the standby, promoted it an new primary
    2. Amazon RDS automatically performs a fail-over when：
      1. Loss of availability in primary Availability Zone.
      2. Loss of network connectivity to primary.
      3. Compute unit failure on primary.
      4. Storage failure on primary.
    3. 修改存储大小需要重启、而设置自动扩容不需要
    4. Amazon RDS Metrics：
      1. regular items：CPU Utilization, Database Connections, Freeable Memory
      2. Enhance items：OS processes, RDS processes, RDS child processes
  7. SQS、SNS、SWF、Step functions
    1. Amazon SQS retention period is from 1 minute to 14 days
    2. Amazon Simple Workflow Service (SWF)：creating a decoupled architecture in AWS；不会重复消费任务；适合ec2，已过时，除非有父子进程信号才用
    3. Step functions：适合lambda、ECS、API Gateway；替代SWF
  8. 防火墙
    1. AWS Firewall Manager：simplifies your AWS WAF and AWS Shield Advanced administration and maintenance tasks across multiple accounts and resources.
    2. Amazon GuardDuty：一种智能威胁检测服务，通过 CloudTrail 日志、VPC 流日志、DNS 日志分析出账户盗用、存储桶入侵、异常 API 调用、恶意 IP 等活动，并自动化响应。
    3. AWS WAF：七层协议的防护（ALB, API Gateway, CloudFront），功能有ACL（封ip、header、url）、SQL注入、XSS、XSS、Rate-based for DDoS。
    4. AWS Shield：三层/四层协议、高级的DDoS防护，标准版免费、高级版可以用于EIP、ELB、CloudFront、Global Accelerator、Route 53。
  9. 存储数据库密码、API密钥的方案：
    1. AWS Secrets Manager：manage database credentials, passwords, API keys, arbitrary text. encrypted and regular rotated.
    2. Systems Manager Parameter Store：可以存明文、密文，但是不会 regular rotated.
  10. 负载均衡器
    1. 同ip多域名https支持(SNI)：CLB不支持，ALB、ELB、CloudFront支持
    2. 支持websocket：ALB支持；ELB仅支持HTTP、HTTPs、TCP、UDP、TLS，可以在ec2层实现websocket，然后用TCP->ws，TLS->wss。
    3. 只有ALB支持 path-based and host-based routing
  11. 加速传输
    1. Snowball：80T，可用多个；如果用100Mb带宽传，得200天
    2. Snowmobile：100P
    3. AWS Storage Gateway：使用S3、FSx等扩展本地机房的存储（不支持EFS）；local caching可以加速访问
    4. AWS DataSync：不能加速，接口比较广泛：S3, EFS, Fsx for Windows
    5. S3 Transfer Acceleration：全球各地的客户需要上传S3
    6. AWS Direct Connect（DX）：需要最少一月的搭建
    7. AWS Site to Site VPN（IPsec）
  12. 加速全球访问
    1. AWS Global Accelerator：给节点（NLB、ALB、EC2、EIP）加速；good fit for gaming(UDP), IoT(MQTT), static IP(Http)
    2. AWS CloudFront
  13. 网络连通
    1. Private Link（VPC Endpoint Services）：把S3、DynamoDB等服务变成私网服务
    2. Direct Connect Gateway：打通一个本地机房到多个 AWS 的区域
    3. AWS VPN CloudHub：打通多个本地机房的连接（依赖VPN或DX）
    4. AWS Transit Gateway：可以理解为云上的路由器，在每一个 Region 中创建一个 TGW，就可以将它们 Peering 起来
  14. 大数据
    1. AWS Kinesis Firehose：是一个消化管道，延迟60秒，不加处理的传给S3、Redshift
    2. AWS Kinesis Streams：是实时和自定义的，延迟1秒，可以自定义处理数据；默认保留数据1天；关键字：real-time analytics
  15. 容灾
    1. active-active：每个节点都是主节点；active-passive：一般都是主节点提供服务，从节点容灾
    2. High Availability：不间断服务；Fault Tolerance：假设需要4个ec2，目前AZ1有2个，AZ2有2个，如果AZ1挂了，虽然是不间断，但是容错能力不够，最好是两个AZ都4个ec2
  16. 费用成本
    1. ec2：不是running状态无需付费，但是stopping到休眠状态需要；预留实例terminated状态也要付费
